# 环境依赖安装说明

## macOS Apple Silicon

macOS 分支当前采用 clone 仓库 + `install.sh` 的安装方式，暂不提供 dmg。

### 必需环境

- macOS 13 及以上
- Apple Silicon（arm64）
- Python 3.13
- `uv`（推荐通过 Homebrew 安装：`brew install uv`）

### 安装流程

```bash
bash install.sh
```

`install.sh` 会自动创建 / 检查项目内 `.venv`、安装 Python 依赖、在当前机器重建 `CapsWriter.app` 启动器，并把 `capswriter` 命令安装到 `~/.local/bin`。

### Python 动态库加载策略

macOS 客户端为了让麦克风权限归属显示为 CapsWriter，会使用嵌入式 CPython launcher。该 launcher 必须链接 `libpython3.13.dylib`，但不能使用开发者机器上的绝对路径。

当前策略：

- `build_launcher.sh` 从 `.venv/bin/python` 读取当前机器的 `sys.base_prefix` 与 `sysconfig`。
- `libpython3.13.dylib` 通过 `.venv/lib/` 下的符号链接暴露给 `.app`。
- Mach-O 只写入相对 rpath：`@executable_path/../../../../.venv/lib`。
- Python base prefix 写入 `.venv/capswriter-python-prefix`，运行时读取，不写入二进制。

因此用户 clone 后只需要执行 `bash install.sh` 或 `bash build_launcher.sh`，不需要手动配置 Python 动态库路径。

### 常见问题

如果日志出现类似 `/Users/某个开发者/.local/share/.../libpython3.13.dylib (no such file)`，说明正在运行旧的、在其他机器上构建过的 launcher。处理方式：

```bash
bash install.sh
capswriter install
capswriter restart
```

也可以先执行：

```bash
capswriter doctor
```

`doctor` 会检查启动器是否仍包含 `/Users/...` 绝对路径，并提示是否需要重建。

## VC++ 运行库

CapsWriter-Offline 依赖 VC++ 运行库。如果启动时提示缺少 DLL，说明系统未安装。

**下载安装**：<https://aka.ms/vs/17/release/vc_redist.x64.exe>

直接下载安装即可，一般安装一次后系统里的所有程序都能用。


## FFmpeg

文件转录功能需要调用 `ffmpeg` 来提取音视频文件的音频流。

安装方式：

1. 访问 <https://ffmpeg.org/download.html>，点击 Windows 下的图标
2. 推荐从 **gyan.dev** 下载：<https://www.gyan.dev/ffmpeg/builds/ffmpeg-git-full.7z>
3. 解压后，将 `ffmpeg/bin` 文件夹路径添加到系统 PATH 环境变量中
